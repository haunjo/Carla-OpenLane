# Carla-OpenLane Dataset Specification

This document describes the structure, format, and usage of the Carla-OpenLane dataset.

---

## Overview

**Carla-OpenLane** is a synthetic dataset for 3D lane topology detection, generated using CARLA simulator and annotated in OpenLane-V2 format.

**Key features:**
- Multi-camera setup (ArgoVerse2 / nuScenes compatible)
- HD map annotations (lane segments, topology, traffic elements)
- Diverse scenarios (10 towns, multiple weather conditions)
- Large-scale (up to 200 scenes per subset)

## Data Format

### 1. Metadata (`{frame}.json`)

Each frame's metadata file contains:

```json
{
  "meta_data": {
    "source_data": "argoverse2",       // or "nuscenes"
    "map_path": "Town01.xodr",
    "town": "Town01",
    "weather": "ClearNoon",
    "traffic_density": "Medium"
  },
  "sensor_data": {
    "CAM_FRONT": {
      "image_path": "../image_2/0.jpg",
      "intrinsic": {
        "K": [[fx, 0, cx], [0, fy, cy], [0, 0, 1]]
      },
      "extrinsic": {
        "rotation": [qw, qx, qy, qz],      // Quaternion
        "translation": [x, y, z]
      }
    },
    "CAM_FRONT_LEFT": { ... },
    "CAM_FRONT_RIGHT": { ... },
    "CAM_BACK": { ... },
    "CAM_BACK_LEFT": { ... },
    "CAM_BACK_RIGHT": { ... },
    "CAM_FRONT_FISHEYE": { ... }          // ArgoVerse2 only
  },
  "ego_pose": {
    "xyz": [x, y, z],                     // Ego vehicle position (world frame)
    "rotation": [roll, pitch, yaw]        // Euler angles (radians)
  },
  "annotation": {
    "traffic_element": [
      {
        "id": 1,
        "category": 1,                    // 1: traffic light, 2: sign
        "attribute": 1,                   // Light state: 1=red, 2=yellow, 3=green
        "points": [[x1, y1, z1], [x2, y2, z2]],
        "affected_lanelet": [101, 102]    // Lanelet2 IDs
      }
    ]
  }
}
```

### 2. Annotations (`{frame}-ls.json`)

OpenLane-V2 format lane segment annotations:

```json
{
  "lane_segment": [
    {
      "id": 0,
      "centerline": [[x1, y1, z1], [x2, y2, z2], ...],  // 3D points in ego frame
      "left_laneline": {
        "points": [[x1, y1, z1], ...],
        "type": "solid"                   // solid, dashed, double, etc.
      },
      "right_laneline": {
        "points": [[x1, y1, z1], ...],
        "type": "dashed"
      },
      "is_intersection": false,
      "topology": {
        "predecessors": [1, 2],           // Connected lane IDs
        "successors": [3, 4]
      }
    }
  ],
  "area": [
    {
      "id": 0,
      "category": 2,                      // 1: crosswalk, 2: road border
      "points": [[x1, y1, z1], ...]       // Polygon vertices
    }
  ],
  "topology_lsls": [                     // N×N lane-to-lane connectivity
    [0, 1, 0, 0],                        // Row i: connections from lane i
    [0, 0, 1, 1],
    ...
  ],
  "topology_lste": [                     // N×M lane-to-traffic-element
    [1, 0, 0],                           // Row i: traffic elements affecting lane i
    [0, 1, 0],
    ...
  ]
}
```

### 3. Split Manifest (`train.json`)

```json
{
  "meta_info": {
    "dataset": "Carla-OpenLane",
    "subset": "argoverse2",
    "split": "train",
    "total_scenes": 200,
    "total_frames": 6000
  },
  "data_list": [
    {
      "scene_id": "0000",
      "frame_id": "0",
      "image_path": "datasets/raw/train/0000/image_2/0.jpg",
      "annotation_path": "datasets/raw/train/0000/info/0-ls.json",
      "meta_path": "datasets/raw/train/0000/info/0.json"
    },
    ...
  ]
}
```

## Coordinate Systems

### 1. World Frame (CARLA Global)

- **Origin:** CARLA map origin
- **Axes:** X=East, Y=North, Z=Up
- **Used in:** OpenDRIVE maps, Lanelet2 maps

### 2. Ego Vehicle Frame

- **Origin:** Ego vehicle center
- **Axes:** X=Forward, Y=Left, Z=Up
- **Used in:** Annotations (`*-ls.json`)

**Transformation:**
```python
# World → Ego
p_ego = R_yaw @ R_pitch @ R_roll @ (p_world - ego_xyz)

# Where:
R_yaw = rotation_matrix_z(yaw)
R_pitch = rotation_matrix_y(pitch)
R_roll = rotation_matrix_x(roll)
```

### 3. Camera Frame

- **Origin:** Camera optical center
- **Axes:** X=Right, Y=Down, Z=Forward
- **Used in:** Image projection

**Projection:**
```python
# Ego → Camera
p_cam = R_cam @ p_ego + t_cam

# Camera → Image
u, v = K @ p_cam / p_cam[2]
```

## Data Preprocessing

### Convert to PKL for LaneSegNet

```bash
cd LaneSegNet/data

# Full annotation
python gt_generator.py \
  --data-json ../../datasets/splits/train.json \
  --output carla_train_ls.pkl

# Centerline only (lightweight)
python gt_generator_centerline.py \
  --data-json ../../datasets/splits/train.json \
  --output carla_train_centerline.pkl
```

**PKL structure:**
```python
{
  'images': List[str],              # Image paths
  'annotations': List[Dict],        # Lane annotations
  'meta_infos': List[Dict],         # Scene metadata
  'ego_poses': List[np.ndarray],    # Ego vehicle poses
  'cam_intrinsics': List[Dict],     # Camera parameters
}
```

## Quality Assurance

### Validation Checklist

- [ ] All frames have corresponding `*-ls.json` files
- [ ] Topology matrices are square (N×N for LSLS)
- [ ] No invalid lane IDs in topology
- [ ] Traffic elements reference valid lanelets
- [ ] Image paths are accessible
- [ ] Camera extrinsics are valid quaternions

### Validation Script

```bash
cd OpenLane-V2-HDmap-Converter
python src/checksum.py \
  --root ../Carla-OpenLane/datasets/raw/train \
  --root ../Carla-OpenLane/datasets/raw/val
```

## Known Issues & Limitations

1. **Scene 0042, 0137:** Missing traffic light annotations (CARLA spawn issue)
2. **Intersection lanes:** May have sparse centerline points due to camera visibility filtering
3. **Night scenes:** Reduced annotation quality in low-light conditions
4. **Town10:** Complex intersections may have incomplete topology

**Workaround:** Filter problematic scenes using `datasets/statistics/invalid_scenes.txt`
